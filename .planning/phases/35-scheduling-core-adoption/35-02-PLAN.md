<objective>
Подключить новый scheduling core к persistence и MCP command surface, чтобы сервер перестал делать raw record-level date rewrites без полноценного cascade.

Purpose: Headless scheduling module сам по себе ничего не решает, пока task mutations и MCP tools не используют его как authoritative execution layer.
Output: Command-aware `TaskService` и MCP tools, возвращающие true scheduling diffs.
</objective>

```ts

  -> load full snapshot
  -> scheduler.execute(command, snapshot)
  -> persist changed tasks transactionally
  -> sync parent summary ranges
  -> return { changedTasks, changedIds, ... }
```
