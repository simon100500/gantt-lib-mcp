<!-- Important: The onChange pattern -->

Per RESEARCH.md:
"Decision: Pass `setTasks` directly to `onChange` prop.
Rationale: gantt-lib emits functional updaters to avoid stale closure bugs.
The library handles the `prev => next` pattern internally."

This means we pass: `onChange={setTasks}` NOT `onChange={(newTasks) => setTasks(newTasks)}`
</context>
