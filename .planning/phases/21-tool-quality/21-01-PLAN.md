# Plan 21-01: Tool Quality Improvements

**Improve MCP tool descriptions and error messages per MCP best practices**

## Objective

Enhance the AI agent's user experience by improving tool descriptions and error messages to be semantic, actionable, and recovery-oriented. This phase focuses on the quality of communication between the MCP server and AI agents, ensuring they can understand tools better and recover from errors independently.

**Purpose:** Tool descriptions are the only UI an AI model sees. Poor descriptions lead to incorrect tool selection. Poor error messages leave the agent stuck without recovery path.

**Output:** 9 active tools with semantic descriptions and cross-references, all errors following "what + why + fix" pattern, 3 legacy tools removed.

## Context

### Project Context
- **Phase:** 21 - Tool Quality (final phase of v3.0 milestone)
- **Status:** Ready to execute
- **Dependencies:** None (can run independently after Phases 17-20)

### User Decisions (from 21-CONTEXT.md)
**Locked Decisions:**
1. **Tool Description Format** - Single sentence summary with structure: what does + key parameters + result
2. **Cross-References** - Add references between related tools (create_task → get_tasks, add_message → get_conversation_history)
3. **Error Message Pattern** - `[Permanent/Temporary] What failed. Why it failed. Fix: concrete action with example`
4. **Legacy Tool Removal** - Delete set_autosave_path, export_tasks, import_tasks (no-op tools)
5. **Update Order** - By frequency: create_task → get_tasks → update_task → get_task/delete_task → create_tasks_batch/get_conversation_history/add_message

**Claude's Discretion:**
- Exact wording of descriptions (must be semantic and dense)
- Error formatting (single line or multi-line)
- Edge case handling (empty strings, null, etc.)

### MCP Best Practices (from MCP-recomendations.md)
- **Section 4:** Tool descriptions are the only UI — must be semantically dense
- **Section 5:** Error handling — model must know what to do next
- **Pattern:** "что + почему + что делать" (what + why + what to do)
- **Semantic density:** Maximum meaning per unit of text
- **Error markers:** [Permanent] vs [Temporary] for retry guidance

### Existing Code Patterns
- Current descriptions: Generic and verbose ("Create a new Gantt chart task with name, dates, and optional properties")
- Current errors: Basic format ("Invalid startDate format: ${input.startDate}. Expected format: YYYY-MM-DD")
- Validation functions: `isValidDateFormat()`, `isValidDateRange()`, `isValidDependencyType()` (lines 47-63)
- Tool definitions: Lines 66-400 in index.ts
- Tool handlers: Lines 400-999 in index.ts
- Legacy tools: set_autosave_path (line 294), export_tasks (line 272), import_tasks (line 280)
