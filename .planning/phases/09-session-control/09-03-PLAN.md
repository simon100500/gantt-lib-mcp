<!-- WS auth handshake pattern (no HTTP header support in browser WS) -->
Client connects → sends { type: 'auth', token: '<accessToken>' } as first message
Server verifies token → stores socket in Map<sessionId, Set<WebSocket>>
Server sends { type: 'connected' } after successful auth
Server sends { type: 'error', message: 'Unauthorized' } + closes socket on auth failure

New internal state:
```typescript
const sessionConnections = new Map<string, Set<WebSocket>>();  // sessionId → sockets
const chatHandlers: Array<(msg: string, sessionId: string) => void> = [];
```

Remove the old `connections` Set.

**Updated broadcast(msg):** iterates all sockets across all sessions (same behavior as before):
```typescript
export function broadcast(msg: ServerMessage): void {
  const json = JSON.stringify(msg);
  for (const sockets of sessionConnections.values()) {
    for (const socket of sockets) {
      if (socket.readyState === 1) socket.send(json);
    }
  }
}
```

**New broadcastToSession(sessionId, msg):**
```typescript
export function broadcastToSession(sessionId: string, msg: ServerMessage): void {
  const sockets = sessionConnections.get(sessionId);
  if (!sockets) return;
  const json = JSON.stringify(msg);
  for (const socket of sockets) {
    if (socket.readyState === 1) socket.send(json);
  }
}
```

**Updated onChatMessage handler signature:**
```typescript
export function onChatMessage(handler: (msg: string, sessionId: string) => void): void
```

**Updated registerWsRoutes with auth handshake:**
The WS route now does NOT immediately add the socket to sessionConnections. Instead:
1. Socket connects → server waits for first message
2. First message must be `{ type: 'auth', token: string }`
3. Server imports verifyToken from './auth.js' and verifies the token
4. On success: extracts sessionId from payload, adds socket to Map, sends { type: 'connected' }
5. On failure: sends { type: 'error', message: 'Unauthorized' }, closes socket
6. Subsequent messages of type 'chat' trigger chatHandlers with (message, sessionId)
7. On close: removes socket from its Set (cleanup the Map entry if Set becomes empty)

Import verifyToken: `import { verifyToken } from './auth.js'`
Import type JwtPayload: `import type { JwtPayload } from './auth.js'`

Keep the WsClientMessage type export for the web client to use.
