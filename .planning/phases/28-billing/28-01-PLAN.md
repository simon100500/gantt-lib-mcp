    ```sql
    CREATE TABLE IF NOT EXISTS subscriptions (
      id TEXT PRIMARY KEY,
      user_id TEXT NOT NULL UNIQUE REFERENCES users(id) ON DELETE CASCADE,
      plan TEXT NOT NULL DEFAULT 'free',
      period_start TEXT,
      period_end TEXT,
      ai_used INTEGER NOT NULL DEFAULT 0,
      created_at TEXT NOT NULL
    )
    ```

    ```sql
    CREATE TABLE IF NOT EXISTS payments (
      id TEXT PRIMARY KEY,
      user_id TEXT NOT NULL REFERENCES users(id) ON DELETE CASCADE,
      plan TEXT NOT NULL,
      period TEXT NOT NULL CHECK(period IN ('monthly', 'yearly')),
      amount REAL NOT NULL,
      yookassa_payment_id TEXT NOT NULL UNIQUE,
      status TEXT NOT NULL DEFAULT 'pending' CHECK(status IN ('pending', 'succeeded', 'failed', 'refunded')),
      created_at TEXT NOT NULL
    )
    ```

    Plan config driven by env variables with defaults matching the tariff table from CONTEXT.md (D-09):

    ```typescript
    export type PlanKey = 'free' | 'start' | 'team' | 'enterprise';

    export interface PlanLimits {
      projects: number;       // -1 = unlimited
      aiGenerations: number;  // -1 = unlimited
      aiRefinements: number;  // per project, -1 = unlimited
      resources: number;      // -1 = unlimited
      teamMembers: number;    // -1 = unlimited
    }

    export interface PlanPricing {
      monthly: number;   // price in rubles
      yearly: number;    // price in rubles
      description: string;
    }
    ```

    Export `PLAN_CONFIG` record keyed by PlanKey with:
    - `limits: PlanLimits` — quotas for each plan from env with defaults:
      - free: projects=2, aiGenerations=3, aiRefinements=5, resources=1, teamMembers=1
      - start: projects=5, aiGenerations=10, aiRefinements=20, resources=20, teamMembers=1
      - team: projects=20, aiGenerations=-1, aiRefinements=-1, resources=-1, teamMembers=5
      - enterprise: projects=-1, aiGenerations=-1, aiRefinements=-1, resources=-1, teamMembers=20
    - `pricing: PlanPricing` — monthly/yearly prices from env with defaults:
      - free: 0/0
      - start: 1490/12000
      - team: 4990/59880
      - enterprise: 12900/154800

    Export function `getPlanLimits(plan: PlanKey): PlanLimits` that returns limits from PLAN_CONFIG.
    Export function `getPlanPricing(plan: PlanKey, period: 'monthly' | 'yearly'): number` that returns price.
    Export function `isPlanActive(periodEnd: string | null): boolean` that checks if ISO date is in the future.

    **3. Create packages/server/src/services/billing-service.ts:**

    Class `BillingService` with these methods:

    - `getOrCreateSubscription(userId: string): Promise<SubscriptionRow>` — SELECT from subscriptions by user_id, if not found INSERT with plan='free', return row
    - `getSubscriptionStatus(userId: string): Promise<{ plan: PlanKey, periodEnd: string | null, aiUsed: number, aiLimit: number, isActive: boolean }>` — get subscription, compute aiLimit from plan config, check if active
    - `applyPlan(userId: string, plan: PlanKey, period: 'monthly' | 'yearly'): Promise<void>` — UPDATE subscriptions SET plan, period_start=now(), period_end=now()+31days or +365days, ai_used=0 (D-01, D-08)
    - `incrementAiUsage(userId: string): Promise<{ used: number, limit: number }>` — increment ai_used, return current used/limit
    - `createPaymentRecord(userId: string, plan: PlanKey, period: 'monthly' | 'yearly', amount: number, yookassaPaymentId: string): Promise<PaymentRow>` — INSERT into payments
    - `markPaymentSucceeded(yookassaPaymentId: string): Promise<PaymentRow | null>` — UPDATE payments SET status='succeeded', return row
    - `getPaymentHistory(userId: string): Promise<PaymentRow[]>` — SELECT from payments WHERE user_id ORDER BY created_at DESC
    - `isPaymentProcessed(yookassaPaymentId: string): Promise<boolean>` — check if payment exists in payments table (for webhook idempotency)

    All methods use `getDb()` from `@gantt/mcp/db`.
    Use `crypto.randomUUID()` for ID generation.
    Use `new Date().toISOString()` for timestamps.

    **1. Create packages/server/src/routes/billing-routes.ts:**

    Follow the `registerXxxRoutes(fastify: FastifyInstance)` pattern from auth-routes.ts.
    Import `authMiddleware` from `../middleware/auth-middleware.js`.
    Import `BillingService` from `../services/billing-service.js`.
    Import `PLAN_CONFIG, getPlanPricing` from `../services/plan-config.js`.

    Use raw `fetch` to YooKassa REST API v3 (NOT an npm SDK — the TypeScript SDK ecosystem for YooKassa is immature; use direct HTTP calls with shop_id:secret Basic Auth).

    YooKassa API:
    - Base URL: `https://api.yookassa.ru/v3`
    - Auth: HTTP Basic with shop_id as username, secret_key as password
    - Headers: `Idempotence-Key: <uuid>` and `Content-Type: application/json`

    Env vars: `YOOKASSA_SHOP_ID`, `YOOKASSA_SECRET_KEY`

    **Endpoints:**

    **POST /api/billing/create** (per D-03 embedded widget, D-04 flow):
    - preHandler: [authMiddleware]
    - Body: `{ plan: 'start' | 'team', period: 'monthly' | 'yearly' }`
    - Validate plan is not 'free' or 'enterprise' (per D-02 enterprise is "contact us")
    - Get price from `getPlanPricing(plan, period)`
    - Create YooKassa payment via POST https://api.yookassa.ru/v3/payments with body:
      ```json
      {
        "amount": { "value": "<price>.00", "currency": "RUB" },
        "capture": true,
        "description": "GetGantt — <plan name> (<period>)",
        "confirmation": { "type": "embedded" },
        "metadata": { "userId": "<userId>", "plan": "<plan>", "period": "<period>" },
        "receipt": {
          "customer": { "email": "<user email>" },
          "items": [{
            "description": "GetGantt — <plan name> (<period>)",
            "quantity": 1,
            "amount": { "value": "<price>.00", "currency": "RUB" },
            "vat_code": 1,
            "payment_mode": "full_payment",
            "payment_subject": "service"
          }]
        }
      }
      ```
    - Parse response, get `id` (yookassa payment id) and `confirmation.confirmation_token`
    - Save payment to DB via `BillingService.createPaymentRecord`
    - Return `{ paymentId: string, confirmationToken: string }`

    **GET /api/billing/status?paymentId=xxx** (for client polling):
    - Query param: paymentId (this is the YooKassa payment id)
    - Fetch payment status from YooKassa: GET https://api.yookassa.ru/v3/payments/<paymentId>
    - Return `{ status: string, paid: boolean, plan: string | null }`

    **POST /api/billing/webhook** (per homeopapa pattern):
    - NO auth middleware (YooKassa calls this directly)
    - Body: `{ event: string, object: { id: string, ... } }`
    - Extract payment_id from `payload.object.id`
    - Check idempotency: `BillingService.isPaymentProcessed(payment_id)` — if yes, return `{ ok: true }`
    - Fetch payment details from YooKassa: GET https://api.yookassa.ru/v3/payments/<payment_id>
    - Check `paid === true` and `status === 'succeeded'`
    - Extract `metadata.userId`, `metadata.plan`, `metadata.period`
    - Call `BillingService.applyPlan(userId, plan, period)`
    - Call `BillingService.markPaymentSucceeded(payment_id)`
    - Return `{ ok: true }`

    **GET /api/billing/subscription** (per D-10):
    - preHandler: [authMiddleware]
    - Get subscription status via `BillingService.getSubscriptionStatus(req.user.userId)`
    - Return `{ plan, periodEnd, aiUsed, aiLimit, isActive, limits: PlanLimits }`

    **GET /api/billing/payments** (per D-10):
    - preHandler: [authMiddleware]
    - Get payment history via `BillingService.getPaymentHistory(req.user.userId)`
    - Return array of payment records
