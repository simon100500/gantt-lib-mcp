### Task 4: Fix Prisma DateTime serialization bugs (Commit: 62c8f84)

**Problem:** Prisma's DateTime serialization caused PostgreSQL UTF-8 encoding errors:
```
PostgresError { code: "22021", message: "invalid byte sequence for encoding \"UTF8\": 0x00" }
```

**Root cause:** Prisma's binary protocol was adding null bytes when serializing JavaScript `Date` objects to PostgreSQL TEXT columns (schema uses TEXT for DateTime fields, not TIMESTAMP).

**Solution:** Used Prisma's `$executeRaw` for all INSERT operations with DateTime fields:
- `createOtp()` - `INSERT INTO otp_codes` with ISO string timestamps
- `findOrCreateUser()` - `INSERT INTO users` with ISO string timestamps
- `createProject()` - `INSERT INTO projects` with ISO string timestamps
- `createSession()` - `INSERT INTO sessions` with ISO string timestamps
- `addMessage()` - `INSERT INTO messages` with ISO string timestamps

This bypasses Prisma's type conversion and uses plain SQL with ISO date strings.

**Code pattern:**
```typescript
// Before (caused UTF-8 error):
await db.otpCode.create({
  data: { id, email, code, expiresAt: new Date(), used: false }
});

// After (works):
await db.$executeRaw`
  INSERT INTO otp_codes (id, email, code, expires_at, used)
  VALUES (${id}, ${email}, ${code}, ${new Date().toISOString()}, false)
`;
```

**Files modified:**

## Technical Notes

### DateTime Handling Architecture

The application uses a mixed DateTime handling strategy:
- **Schema:** Prisma schema defines `DateTime` fields
- **Migration:** Creates TEXT columns (PostgreSQL `TEXT` type)
- **Reads:** Prisma returns `Date` objects which we convert to ISO strings
- **Writes:** Use `$executeRaw` with ISO date strings to avoid UTF-8 bugs

This works but is not ideal. The proper fix would be to use `TIMESTAMP` columns in PostgreSQL.

### JWT Verification for EventSource

EventSource API doesn't support custom headers, so JWT must be passed via query parameter:
```typescript
// Client (useTaskStream.ts)
const url = `/stream/tasks?token=${encodeURIComponent(token)}`;
const eventSource = new EventSource(url);

// Server (sse.ts)
const queryToken = (req.query as { token?: string }).token;
if (queryToken) {
  payload = verifyToken(queryToken);
}
```
