# Plan 13.1-01: SSE Architecture Migration

**Objective:** Replace WebSocket architecture with Server-Sent Events (SSE) and migrate from SQLite to PostgreSQL with NOTIFY/LISTEN triggers for real-time updates.

**Purpose:** SSE is more suitable for one-way server-to-client streaming (AI responses, task updates) and simplifies the architecture. PostgreSQL NOTIFY/LISTEN provides database-level real-time notifications without additional application code.

**Output:** Working SSE endpoints, PostgreSQL schema with triggers, client hooks for SSE streams, updated Nginx configuration.

### Existing Patterns

**WebSocket Registry (to be replaced):**
- `Map<sessionId, Set<WebSocket>>` for session-isolated connections
- `broadcastToSession()` sends to specific session's sockets
- Auth handshake: first message must be `{ type: 'auth', token }`

**Client WebSocket (to be replaced):**
- `useWebSocket` with exponential backoff reconnection
- Token-reactive reconnect via `accessToken` dependency

**Database Schema (to migrate to Prisma + PostgreSQL):**
- Tables: users, projects, sessions, otp_codes, tasks, dependencies, messages
- SQLite via @libsql/client
- Foreign key CASCADE deletes
