    Create `packages/mcp/src/services/command.service.ts` with a `CommandService` class.

    The `commitCommand` method flow:
    1. Start a Prisma `$transaction` with `{ isolationLevel: 'Serializable' }` (or at minimum, use interactive transaction with version check)
    2. Load current Project with `version` field inside the transaction
    3. If `project.version !== request.baseVersion` → return rejected response: `{ clientRequestId, accepted: false, reason: 'version_conflict', currentVersion: project.version }`
    4. Load full task snapshot for projectId (all tasks + dependencies)
    5. Convert to `Task[]` array compatible with gantt-lib
    6. Execute command through gantt-lib:
       - `move_task` → `moveTaskWithCascade(taskId, new Date(startDate + 'T00:00:00.000Z'), snapshot, opts)`
       - `resize_task` → `resizeTaskWithCascade(taskId, anchor, new Date(date + 'T00:00:00.000Z'), snapshot, opts)`
       - `recalculate_schedule` with taskId → `recalculateTaskFromDependencies(taskId, snapshot, opts)`
       - `recalculate_schedule` without taskId → `recalculateProjectSchedule(snapshot, opts)`
       - `set_task_start` → compute new dates, call `moveTaskWithCascade`
       - `set_task_end` → compute new dates, call `resizeTaskWithCascade(taskId, 'end', ...)`
       - `change_duration` → compute new endDate from duration, call `resizeTaskWithCascade`
       - `create_task` → create task via Prisma, then recalculate affected tasks
       - `delete_task` → delete task + dependencies via Prisma, then recalculate
       - `create_dependency`, `remove_dependency`, `change_dependency_lag` → update dependency via Prisma, recalculate
       - `reparent_task` → update parentId via Prisma, recalculate parents
       - `reorder_task` → update sortOrder via Prisma (no scheduling change)
    7. Compute patches by comparing before/after for each changed task:
       - `reason: 'direct_command'` if task.id === command target
       - `reason: 'parent_rollup'` if task has children
       - `reason: 'dependency_cascade'` otherwise
    8. Persist all changed tasks to DB (update rows in transaction)
    9. Bump version: `prisma.project.update({ where: { id: projectId, version: baseVersion }, data: { version: { increment: 1 } } })`
    10. Create ProjectEvent record with command, result, patches, executionTimeMs
    11. Load final snapshot
    12. Return accepted response with `{ clientRequestId, accepted: true, baseVersion, newVersion: baseVersion + 1, result: { snapshot, changedTaskIds, patches, conflicts, changedDependencyIds: [] }, snapshot }`

    Helpers needed:
    - `getScheduleOptions(projectId)` → load project's ganttDayMode, return `{ businessDays: mode === 'business' }`
    - `loadTaskSnapshot(projectId, prismaClient)` → query all tasks + dependencies, return `Task[]`
    - `buildProjectSnapshot(tasks, prismaClient, projectId)` → return `ProjectSnapshot`
    - `computePatches(before, after, targetTaskId)` → compute `Patch[]`

    Create `packages/server/src/routes/command-routes.ts` following the pattern of existing route files (see `auth-routes.ts` and `billing-routes.ts` for structure):

    ```typescript
    import type { FastifyInstance } from 'fastify';
    import { commandService } from '@gantt/mcp/services';
    import type { CommitProjectCommandRequest, ActorType } from '@gantt/mcp/types';
    import { authMiddleware } from '../middleware/auth-middleware.js';

    export async function registerCommandRoutes(fastify: FastifyInstance) {
      fastify.post('/api/commands/commit', { preHandler: [authMiddleware] }, async (req, reply) => {
        const body = req.body as Partial<CommitProjectCommandRequest>;

        if (!body.command || !body.clientRequestId || body.baseVersion === undefined) {
          return reply.status(400).send({
            clientRequestId: body.clientRequestId ?? '',
            accepted: false,
            reason: 'validation_error',
            currentVersion: -1,
          });
        }

        const request: CommitProjectCommandRequest = {
          projectId: req.user!.projectId,
          clientRequestId: body.clientRequestId,
          baseVersion: body.baseVersion,
          command: body.command,
        };

        const actorType: ActorType = 'user';
        const actorId = req.user!.userId;

        try {
          const response = await commandService.commitCommand(request, actorType, actorId);
          const statusCode = response.accepted ? 200 : (response.reason === 'version_conflict' ? 409 : 400);
          return reply.status(statusCode).send(response);
        } catch (error) {
          fastify.log.error(error, 'Command commit failed');
          return reply.status(500).send({
            clientRequestId: request.clientRequestId,
            accepted: false,
            reason: 'validation_error',
            currentVersion: -1,
          });
        }
      });
    }
    ```
