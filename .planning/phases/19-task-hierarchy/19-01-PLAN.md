# Plan 19-01: Add parentId Filter to get_tasks

**Goal:** Enable `get_tasks` to filter by parent task ID for hierarchical task queries

**Requirements:** HIER-03 (get_tasks supports filtering by parentId?: string | null)

## Context Summary

**HIER-01 and HIER-02 are already complete:**
- `create_task` accepts `parentId?: string` parameter ✓
- `update_task` accepts `parentId?: string | null` parameter ✓
- Parent date recalculation works ✓
- Circular reference detection (A→A) works ✓

**What's missing:** HIER-03 — `get_tasks` cannot filter by `parentId`

## Tasks

### Task 1: Update TaskService.list() to accept parentId parameter

**<read_first>**
- packages/mcp/src/services/task.service.ts (lines 215-257 for current list() implementation)

**<action>**
Update the `list()` method signature to add `parentId?: string | null` parameter after `projectId`:

```typescript
async list(
  projectId?: string,
  parentId?: string | null,
  limit: number = 100,
  offset: number = 0,
  full: boolean = false
): Promise<{ tasks: Task[]; hasMore: boolean; total }>
```

Update the Prisma query to filter by `parentId`:
- If `parentId === null`: filter for root tasks (`where: { parentId: null }`)
- If `parentId === undefined`: no filter (all tasks)
- If `parentId === string`: filter for direct children (`where: { parentId }`)

Combine with existing `projectId` filter in the where clause:

```typescript
const whereClause: any = {};
if (projectId) whereClause.projectId = projectId;
if (parentId !== undefined) whereClause.parentId = parentId;
```

**<acceptance_criteria>**
- `packages/mcp/src/services/task.service.ts` contains `async list(` with `parentId?: string | null` parameter
- Prisma query uses combined where clause with projectId and parentId conditions
- `parentId: null` filters to root tasks only
- `parentId: "task-id"` filters to direct children of that task
- `parentId: undefined` (omitted) returns all tasks

## Verification Criteria

**must_haves:**
1. `get_tasks(parentId=null)` returns only root tasks (WHERE parentId IS NULL)
2. `get_tasks(parentId="task-123")` returns only direct children of task-123
3. `get_tasks()` (without parentId) returns all tasks (backward compatible)
4. Filter applies before pagination (limit/offset affect filtered results)

**Integration tests to verify:**
- Root tasks query: `get_tasks(parentId=null, limit=100)` → only tasks without parent
- Children query: `get_tasks(parentId="some-id", limit=50)` → only children of that parent
- All tasks: `get_tasks(limit=100)` → all tasks (parentId undefined)
- Combined filters: `get_tasks(projectId="xyz", parentId=null)` → root tasks for specific project
