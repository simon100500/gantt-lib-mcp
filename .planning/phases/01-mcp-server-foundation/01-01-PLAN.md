<task type="auto">
  <name>Create MCP server with stdio transport and placeholder tool</name>
  <files>src/index.ts</files>
  <action>
    Create src/index.ts with MCP server initialization:

    1. Import Server class:
       import { Server } from '@modelcontextprotocol/sdk/server/index.js';
       import { StdioServerTransport } from '@modelcontextprotocol/sdk/server/stdio.js';
       import { CallToolRequestSchema, ListToolsRequestSchema } from '@modelcontextprotocol/sdk/types.js';

    2. Create server instance:
       const server = new Server(
         { name: 'gantt-lib-mcp-server', version: '0.1.0' },
         { capabilities: { tools: {} } }
       );

    3. Register list tools handler:
       server.setRequestHandler(ListToolsRequestSchema, async () => ({
         tools: [
           {
             name: 'ping',
             description: 'A simple ping tool to test MCP server connectivity',
             inputSchema: {
               type: 'object',
               properties: {},
             },
           },
         ],
       }));

    4. Register call tool handler:
       server.setRequestHandler(CallToolRequestSchema, async (request) => {
         if (request.params.name === 'ping') {
           return { content: [{ type: 'text', text: 'pong' }] };
         }
         throw new Error('Unknown tool');
       });

    5. Start server with stdio transport:
       async function main() {
         const transport = new StdioServerTransport();
         await server.connect(transport);
         // Server runs via stdio, no explicit listen needed
       }
       main().catch(console.error);

    DO NOT use express/http - MCP uses stdio for CLI integration.
    DO NOT forget to handle errors in main().
  </action>
  <verify>
    1. npm run build compiles without errors
    2. npm run start starts server (will wait for stdin)
    3. Ctrl+C terminates server cleanly
  </verify>
  <done>
    src/index.ts compiles, starts without errors, and waits for stdio input.
  </done>
</task>
