# Phase 5: batch-tasks - Context

**Gathered:** 2026-02-25
**Status:** Ready for planning

<domain>
## Phase Boundary

Создание MCP инструмента для массового создания задач по шаблону. Инструмент должен поддерживать параметры повторяемости (секции, этажи и т.п.), автоматическое создание связей и работу с параллельными потоками.

**Пример использования:** Создать поэтажный график работ для 6 секций по 4 этажа с 7 видами работ (168 задач) одним вызовом вместо 168 отдельных.

</domain>

<decisions>
## Implementation Decisions

### Двухэтапный подход
- Сначала создать базовые задачи по видам работ с длительностями
- Отдельный вызов для размножения задач по параметрам (секции, этажи)
- Пользователь может комбинировать этапы по необходимости

### Автоматические связи
- FS (Finish-Start) связи между задачами одного вида работ
- Последовательный поток: этаж 1 → этаж 2 → этаж 3 ...
- Связи создаются автоматически на основе порядка задач

### Организация потоков
- Задачи помечаются тегом потока (например, "stream-1", "stream-2")
- Связи создаются только внутри потока
- Разные потоки не зависят друг от друга (могут выполняться параллельно)

### Формирование названий
- Отдельные поля: `workType`, `section`, `floor` (или другие параметры повторяемости)
- Название собирается автоматически: `{workType} {section} секция {floor} этаж`
- Шаблон названия настраивается

### Задание дат
- Указывается `startDate` только для первой задачи в потоке
- Длительности задач задаются явно (в днях)
- Последующие задачи вычисляются автоматически: start = previous.end + 1

### Обработка ошибок
- Частичный успех: создаём задачи пока можем
- При ошибке возвращается список созданных и неудачных задач
- Нет полного отката при ошибке одной задачи

### Claude's Discretion
- Точная структура JSON для входных данных
- Имена полей для параметров повторяемости
- Формат ответа при частичном успехе
- Валидация входных данных

</decisions>

<specifics>
## Specific Ideas

**Пример запроса:**
```
{
  "baseStartDate": "2026-03-01",
  "workTypes": [
    {"name": "Бетонирование стен", "duration": 10},
    {"name": "Бетонирование перекрытий", "duration": 10},
    ...
  ],
  "repeatBy": {
    "sections": [1, 2, 3, 4, 5, 6],
    "floors": [1, 2, 3, 4]
  },
  "streams": 2
}
```

**Название инструмента:** `create_tasks_batch`

**Связи:** Задачи одного вида работ связываются последовательно по этажам внутри каждой секции и потока.

</specifics>
