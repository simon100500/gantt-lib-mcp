<objective>
Fix critical API bug where single task changes send entire task array to server, causing 500 errors and lost changes. Replace bulk PUT /api/tasks with individual task operations (PATCH, POST, DELETE).

Purpose: Current architecture causes DELETE ALL + INSERT ALL on every drag/indent operation, leading to race conditions, lost changes, and poor scalability. This refactor fixes the blocking user workflow issue.

Output: Individual task API endpoints (PATCH /api/tasks/:id, POST /api/tasks, DELETE /api/tasks/:id), frontend mutation hook, updated Gantt component using granular operations
</objective>
