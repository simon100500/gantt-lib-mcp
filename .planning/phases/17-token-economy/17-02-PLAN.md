  Modify MessageService.list() method to support limiting the number of messages returned:

  **Current implementation (line 53-60):**
  ```typescript
  async list(projectId: string): Promise<Message[]> {
    const messages = await this.prisma.message.findMany({
      where: { projectId },
      orderBy: { createdAt: 'asc' },
    });

    return messages.map(m => this.messageToDomain(m));
  }
  ```

  **Updated implementation:**
  ```typescript
  async list(projectId: string, limit: number = 20): Promise<Message[]> {
    // Fetch last N messages ordered by creation time (most recent first)
    const messages = await this.prisma.message.findMany({
      where: { projectId },
      orderBy: { createdAt: 'desc' }, // Changed to desc to get most recent first
      take: limit, // Take only the last N messages
    });

    // Reverse to maintain chronological order (oldest first)
    return messages.reverse().map(m => this.messageToDomain(m));
  }
  ```

  **Key changes:**
  1. Add `limit: number = 20` parameter with default value of 20
  2. Change `orderBy` from `asc` to `desc` to fetch most recent messages first
  3. Add `take: limit` to limit query results
  4. Add `.reverse()` before mapping to restore chronological order (oldest first)
  5. No validation needed (Prisma handles limit automatically, negative values return empty array)

  **Behavior:**
  - `list(projectId)` returns last 20 messages in chronological order
  - `list(projectId, 50)` returns last 50 messages
  - `list(projectId, 10)` returns last 10 messages
  - No error thrown for invalid limits (Prisma handles edge cases)

  **No truncation notice:** Per CONTEXT.md decision, truncate silently without notice to agent.
