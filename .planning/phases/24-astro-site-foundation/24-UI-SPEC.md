# Phase 24 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Design System

| Property | Value |
|----------|-------|
| Tool | Manual (Astro + Tailwind CSS) |
| Preset | not applicable |
| Component library | Radix UI (via React islands) |
| Icon library | lucide-react |
| Font | Noto Sans (400, 600, 700, 800) |

**Rationale:** Astro site использует React islands для интерактивности. UI компоненты переиспользуются из packages/web/src/components/ui/ с адаптацией для Astro. Noto Sans заменяет Roboto для лучшей поддержки кириллицы на маркетинговом сайте.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing |
| md | 16px | Default element spacing |
| lg | 24px | Section padding |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

**Exceptions:**
- Touch targets: minimum 44px for mobile buttons (accessibility)
- Hero padding: 96px top, 72px bottom (from HTML reference)

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 | 1.5 |
| Label | 16px | 600 | 1.4 |
| Heading h3 | 24px | 700 | 1.2 |
| Heading h2 | 32px | 700 | 1.2 |
| Display h1 | 48-84px | 800 | 1.0 |

**Display specifics:**
- Hero h1: `clamp(48px, 7vw, 84px)` — fluid scaling
- Letter-spacing: -3px for hero (tight, modern look)
- Font family: Noto Sans (cyrillic support)

**Rationale:** 4 size levels provide sufficient hierarchy without cognitive overload. Line-height 1.5 for body ensures readability, 1.2 for headings creates tight, modern look typical for SaaS landing pages.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | hsl(210 20% 98%) / #fafaf8 | Background, surfaces |
| Secondary (30%) | hsl(0 0% 100%) / #ffffff | Cards, input demo, gantt preview |
| Accent (10%) | hsl(217 91% 60%) / #1d4ed8 | Primary CTA buttons, logo square, rotating words, interactive elements |
| Destructive | hsl(0 72% 51%) / #ef4444 | Not used in this phase |

**Accent reserved for:**
- Primary CTA button "Попробовать бесплатно"
- Logo square background
- Rotating value prop words (текста, сметы, ТЗ, etc.)
- Typing cursor
- "Создать Гантт" button in demo
- Interactive hover states (nav links, buttons)

**Supporting colors:**
- Text primary: hsl(222 47% 11%) / #18181b
- Text secondary: hsl(240 4% 46%) / #52525b
- Muted: hsl(240 5% 65%) / #a1a1aa
- Border: hsl(210 14% 90%) / #e4e2dc
- Accent-bg: hsl(217 91% 97%) / #eff4ff (generated badge background)

**Rationale:** 60/30/10 split creates clear visual hierarchy. Dominant neutral background lets content breathe. White cards create focal points. Indigo accent (#1d4ed8) matches HTML reference and provides strong CTAs without overwhelming.

## Animation Contract

### Rotating Value Props
- **Pattern:** Slide up/down with fade
- **Duration:** 420ms transition (cubic-bezier(0.4, 0, 0.2, 1))
- **Interval:** 2500ms between rotations
- **Width animation:** 350ms for container width change
- **Words:** текста, сметы, брифа, ТЗ, письма, таблицы (6 variants, circular)

### Typing Demo
- **Pattern:** Typewriter → erase → next text
- **Typing speed:** 28ms ± 22ms random per character (natural feel)
- **Pause at end:** 2200ms before erasing
- **Erase speed:** 12ms per character
- **Pause between texts:** 400ms
- **Cursor blink:** 1s step-end infinite

### Gantt Preview
- **Pattern:** Grow from left (scaleX)
- **Duration:** 600ms ease-out
- **Stagger:** 0.15s delays between bars (0.5s, 0.65s, 0.8s, 0.95s, 1.1s)
- **Transform origin:** left edge

### Hero Entrance
- **Pattern:** Fade-up (translateY + opacity)
- **Duration:** 500ms ease-out
- **Stagger:** 0ms (tag), 70ms (title), 140ms (sub), 210ms (actions), 280ms (proof)

### Timer
- **Pattern:** Count up 0.0s → 1.4s
- **Interval:** 100ms
- **Stop at:** 1.4s

**Reduced motion:** All animations respect `prefers-reduced-motion` — disable or simplify to fade-only.

**Rationale:** Typing speed 28ms ± 22ms creates natural, human-like feel (from HTML reference). Staggered entrances guide attention sequentially. 2500ms rotation balances visibility vs engagement.

## Layout Structure

### Mobile (default, < 768px)
- Stacked layout
- Hero padding: 48px 16px
- Font sizes: clamp() scales down
- Nav: hamburger menu only
- CTAs: stacked vertical

### Tablet (768px - 1024px)
- Hero padding: 72px 32px
- Nav: logo left, links center (hide on mobile), CTA right
- CTAs: horizontal flex

### Desktop (> 1024px)
- Hero padding: 96px 32px 72px
- Max-width: 900px (hero), 700px (demo cards)
- Centered layout with horizontal CTAs

**Breakpoints:**
- Mobile: < 768px
- Tablet: 768px - 1024px
- Desktop: > 1024px

**Rationale:** Mobile-first approach (CONTEXT.md D-10). Responsive font sizing via clamp() ensures readability at all viewport sizes. Max-width containers prevent excessive line length on large screens.

## Responsive Behavior

### Navigation
- **Desktop (> 1024px):** Full nav bar with logo, links, CTAs visible
- **Tablet (768px - 1024px):** Same as desktop, reduced padding
- **Mobile (< 768px):** Logo left, hamburger right → drawer on click

### Hero
- **Desktop:** h1 at 84px, full rotating value props visible
- **Tablet:** h1 scales to ~64px via clamp()
- **Mobile:** h1 scales to ~48px, value props wrap if needed

### Demo Cards
- **Desktop:** Max-width 700px, centered
- **Tablet:** Max-width 700px, reduced horizontal padding
- **Mobile:** Full-width with 16px padding, cards touch edges

### CTAs
- **Desktop:** Horizontal flex row
- **Mobile:** Stacked vertical column, primary CTA first

**Rationale:** Ensures usability across devices. Mobile drawer is standard pattern (CONTEXT.md D-11). Stacked CTAs on mobile prevent cramped touch targets.

## Accessibility

### Touch Targets
- Minimum 44px × 44px for all interactive elements
- Buttons: padding 14px 28px (lg), 7px 16px (default)

### Keyboard Navigation
- Tab order: logo → nav links → CTAs → demo inputs
- Focus visible: 2px ring on all interactive elements
- Skip links: Not required (single-page hero)

### Screen Reader
- Rotating words: `aria-live="polite"`, `aria-label="Ротация примеров"`
- Typing demo: `aria-label="Демонстрация ввода текста"`
- Timer: `aria-live="off"` (decorative)

### Color Contrast
- Primary text on background: WCAG AA (4.5:1)
- Primary CTA (white on indigo): WCAG AAA (7:1)
- Secondary text: WCAG AA (4.5:1)

**Rationale:** 44px touch targets exceed WCAG minimum (44×44px). Focus ring inherited from Tailwind config (ring-offset-background). Color contrast verified via HSL values.

### CSS Variable Migration

- `--background`, `--foreground`, `--card`, `--primary`, etc.
- Add `--accent: 217 91% 60%` (indigo from HTML reference)
- Add `--accent-bg: 217 91% 97%` (light indigo for badges)

### Tailwind Config
Reuse `packages/web/tailwind.config.js`:
- Colors: extend with HSL variables
- Spacing: default 4px base unit
- Border radius: `--radius: 0.375rem`
- Fonts: add Noto Sans family

### React Islands
- InputDemo.tsx: Typewriter animation (28ms ± 22ms)
- GanttPreview.tsx: Grow-in animation (600ms, staggered)
- Both use `client:load` or `client:visible` directive

### Performance
- Lazy load React islands with `client:visible`
- Preload Noto Sans font
- Minimize JavaScript (typing demo ~100 lines)

**Rationale:** File structure follows Astro best practices. CSS variables ensure consistency with app. React islands isolated to interactive components only. Lazy loading reduces initial bundle size.
