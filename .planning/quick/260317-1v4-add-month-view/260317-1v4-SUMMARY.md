# Phase quick-260317-1v4 Plan 01: Add Month View Mode Summary

Added month view mode support to the Gantt chart application, exposing the third view mode that the gantt-lib library already supports.

## One-Liner

Added third view mode button ("Месяц") to toggle day width to 2px per day for monthly timeline visualization.

## Changes Made

### Task 1: Update GanttChart viewMode type
**Commit:** `8462cb5`

Updated the `GanttChartProps` interface to accept 'month' as a valid view mode:
- Changed `viewMode?: 'day' | 'week'` to `viewMode?: 'day' | 'week' | 'month'`
- File: `packages/web/src/components/GanttChart.tsx`

### Task 2: Update App.tsx with month button and logic
**Commit:** `83eea62`

Made three changes to `packages/web/src/App.tsx`:

1. **State type update:** Changed viewMode state type from `'day' | 'week'` to `'day' | 'week' | 'month'`

2. **UI addition:** Added third button to view mode toggle with label "Месяц"
   - Placed after "Неделя" button
   - Uses same styling pattern as day/week buttons
   - Active state shows secondary background color

3. **Day width calculation:** Extended ternary operator to handle month view
   - Day view: 24px per day
   - Week view: 8px per day
   - Month view: 2px per day

## Technical Notes

**Day width rationale:**
- 24px/day (day view): Full detail for daily tasks
- 8px/day (week view): Balanced view for weekly planning
- 2px/day (month view): Condensed view for monthly timelines

The gantt-lib library (v0.17.0) already supported month view mode, but the UI only exposed day and week buttons. This change completes the UI to match library capabilities.
