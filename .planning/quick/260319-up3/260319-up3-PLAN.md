Добавить в интерфейс UseAuthResult новый метод:

```typescript
refreshProjects(): Promise<void>;
```

Реализовать метод в create<AuthStore>:
```typescript
async refreshProjects() {
  const token = get().accessToken;
  if (!token) {
    return;
  }

  try {
    const projects = await fetchProjects(token);
    const state = get();
    const nextProjects = projects.length > 0 ? projects : state.projects;

    persistStoredAuth({
      accessToken: state.accessToken,
      refreshToken: getRefreshToken(),
      user: state.user,
      project: state.project,
      projects: nextProjects,
    });

    set({ projects: nextProjects });
  } catch (error) {
    console.error('Failed to refresh projects:', error);
  }
},
```

Разместить после метода createProject (после строки 471). Метод должен использовать существующую функцию fetchProjects (строка 165) и persistStoredAuth (строка 84).

Добавить useEffect после строки 25 (после объявления auth) для автоматической загрузки проектов при монтировании:

```typescript
useEffect(() => {
  if (!auth.isAuthenticated || !auth.accessToken || hasShareToken) {
    return;
  }

  void auth.refreshProjects();
}, [auth.isAuthenticated, auth.accessToken, hasShareToken]);
```

Этот эффект:
1. Проверяет, что пользователь авторизован (auth.isAuthenticated)
2. Проверяет наличие токена доступа (auth.accessToken)
3. Проверяет, что это не shared режим (hasShareToken)
4. Вызывает auth.refreshProjects() для загрузки актуального списка проектов

Разместить после строки 38 (после объявления hasShareToken), перед использованием auth в других эффектах.
