# Современный SaaS лендинг для проекта ГетГант

Из текста в Гант за 30 секунд



---

## Лендинг для ИИ-сервиса графиков Гантта

### Главная идея позиционирования

Не продавайте «ИИ». Продавайте результат — **готовый график из того что уже есть у пользователя**. Смета, ТЗ, письмо, список задач в заметках — всё это уже лежит у него. Он просто не знает что это можно превратить в график за 30 секунд.

Заголовок должен говорить про вход и выход, не про технологию:
```
Из сметы — в график Гантта за 30 секунд
```
Не:
```
ИИ-планировщик нового поколения
```

---

### Hero section

**Бейдж сверху** — короткий, про новизну подхода:
```
Новый способ планировать проекты
```

**Заголовок** — с вращающимся словом источника:
```
Из [текста / сметы / брифа / ТЗ / письма] — в Гантт за 30 секунд
```
Слово меняется каждые 2-3 секунды. Пользователь видит своё — строитель видит «сметы», айтишник видит «ТЗ».

**Подзаголовок** — одно предложение про механику:
```
Вставьте любой документ или опишите проект своими словами — 
получите готовый график с задачами, сроками и командой.
```

**Кнопки:**
- Primary: «Попробовать бесплатно →»
- Ghost: «Смотреть демо» — ведёт на якорь к демо-блоку на этой же странице

**Social proof** — аватарки + звёзды + конкретная цифра команд.

---

### Демо-блок — самое важное

Прямо под hero — живая демонстрация. Не скриншот, не видео. Интерактивный блок из двух карточек:

**Карточка 1 — ввод:**
Имитация текстового поля где печатается реальный пример — смета на ремонт, ТЗ на разработку, описание проекта. Текст набирается и стирается в цикле. Снизу кнопка «Создать Гантт».

**Стрелка вниз** с анимацией — визуально соединяет вход и выход.

**Карточка 2 — результат:**
Мини-Гантт который «вырастает» — бары появляются один за другим с задержкой. Шапка с таймером «сгенерировано за 1.4 с». Футер со статистикой: сколько этапов, задач, исполнителей.

Это и есть магический момент — пользователь видит трансформацию прямо на лендинге.

---

### Блок «Как это работает» — 3 шага

Не список фич. Конкретная цепочка действий:

```
1. Вставьте что есть
   Смета, ТЗ, письмо, таблица Excel, 
   голосовое описание — любой формат.

2. Получите график
   Задачи, зависимости, критический путь, 
   распределение по команде — автоматически.

3. Редактируйте и делитесь
   Перетащите сроки, добавьте исполнителей, 
   отправьте ссылку — всё онлайн.
```

Три блока в одну строку с разделителями → между ними.

---

### Блок болей — для кого это

Три колонки под типы пользователей. Каждая — боль и решение:

**Строители и подрядчики**
```
Было: смета в Excel, график в голове, 
      календарный план вручную на 40 страниц.
Стало: загрузил смету — получил ППР за минуту.
```

**IT-команды и агентства**
```
Было: Jira слишком сложно, Notion не умеет 
      зависимости, Excel не пересчитывает сроки.
Стало: описал спринты — график готов, 
       при изменении одной задачи всё пересчитывается.
```

**Руководители проектов**
```
Было: час на составление плана для клиента, 
      потом переделывать после каждой правки.
Стало: клиент прислал правки — 
       сказал ИИ, получил новый план.
```

---

### Блок фич — только то что отличает

Не перечисляйте всё. Только то чего нет у конкурентов:

**ИИ-генерация из любого текста** — не шаблоны, а реальный анализ документа. Это главная фича, ей первый блок и больше всего места.

**Пересчёт при изменениях** — двинул одну задачу, зависимые пересчитались сами. Показать это анимацией прямо в блоке.

**Доступ по ссылке** — никакой установки, никакого «скачайте приложение». Открыл браузер — работает. Это важно для строителей на планшете на объекте.

**Динамические поля** — для строителей: корпус, этаж, бригада. Для айтишников: спринт, релиз, среда. Пользователь сам добавляет нужные поля, они становятся доступны для всех задач проекта.

**Что не писать в фичах:** «интуитивный интерфейс», «гибкие настройки», «мощный функционал» — это ничего не значит.

Notion  Jira  Linear  Asana  Excel / CSV

Импортируйте откуда угодно. Работайте онлайн.

```

Акцент на PDF-экспорт — для строителей это критично, они сдают бумажные ППР.

Первый график — бесплатно

### Чего не должно быть на этом лендинге

- Объяснений что такое график Гантта — целевая аудитория знает
- Сравнительных таблиц с конкурентами — слишком рано
- Раздела «О компании» и фото команды — не то место
- Блока с отзывами в виде длинных цитат — только имя + должность + одна строка
- Анимированных иконок фич которые крутятся сами по себе
- Слова «ИИ» или «искусственный интеллект» в заголовке — только в описании механики

## Философия

Лендинг — это не страница с информацией. Это воронка из одного экрана. Пользователь должен понять продукт, почувствовать его и захотеть попробовать — до того как прокрутит вниз.

**Одно правило:** каждый элемент либо объясняет, либо убеждает, либо ведёт к действию. Всё остальное — удалить.

---

## Визуальная философия

**Светлая тема, тёплый белый** — не #ffffff, а #fafaf8 или #f7f6f3. Чистый белый выглядит дёшево.

**Один акцентный цвет** — и только один. Синий, зелёный, оранжевый — неважно. Важно что он один и используется только для главного действия и ключевого слова в заголовке.

**Типографика важнее картинок** — крупный заголовок с tight letter-spacing (-2px… -4px) и font-weight 800 работает лучше любой иллюстрации. Шрифт без засечек, с поддержкой кириллицы.

**Пространство — это тоже дизайн** — не бойтесь пустоты. Padding между блоками от 80px до 120px. Элементы дышат.

**Границы вместо теней** — тонкий border 1px rgba(0,0,0,0.08) выглядит современнее чем box-shadow. Тени только там где нужна реальная глубина.

```

Всё по центру. Никаких колонок в hero.

---

## Анимации — принципы

**Меньше — лучше.** Одна хорошо сделанная анимация лучше десяти посредственных.

**Появление при загрузке** — все элементы hero появляются снизу вверх (translateY + opacity) с нарастающим delay по 60-80ms. Создаёт ощущение живости без суеты.

**Интерактив должен быть немедленным** — hover-эффекты не дольше 150ms. Кнопки поднимаются на 2px с лёгкой тенью.

**Одна WOW-анимация** — что-то одно что запоминается. Вращающееся слово, печатная машинка, живой граф. Не три сразу — одно.

**Никакого параллакса** — выглядит устаревшим и вызывает укачивание.

## Интерактивный демо-блок

Самое важное после заголовка — показать продукт в действии прямо на лендинге. Не скриншот, не видео — живой интерфейс.

Правила:
- Выглядит как настоящий продукт, не как иллюстрация
- Реагирует на действия пользователя
- Показывает результат, а не процесс
- Умещается в один экран

Технически: чистый HTML/CSS/JS без фреймворков. Браузерный chrome сверху (три цветные точки + URL) сразу сигнализирует — это интерфейс, не картинка.

---

## Типографическая система

```
Заголовок H1:   clamp(44px, 7vw, 84px)  weight 800  spacing -3px
Заголовок H2:   clamp(28px, 4vw, 48px)  weight 800  spacing -2px
Подзаголовок:   17-18px  weight 400  line-height 1.65  color muted
Интерфейс UI:   12-13px  weight 600  uppercase + letter-spacing для лейблов
Моно / код:     JetBrains Mono или Noto Sans Mono — для дат, цифр, URL
```

---

## Карточки и блоки

Единый радиус — выберите один: 10px, 12px или 16px. Используйте везде.

Карточки строятся из трёх элементов:
```
border: 1px solid var(--border)
border-radius: выбранный радиус
background: white
```

Никаких градиентных фонов на карточках. Никаких цветных блоков на весь экран — только белое на тёплом сером фоне.

Hover на карточке: `border-color → accent` + `transform translateY(-2px)` + лёгкая тень. Ничего больше.

---

## Цветовая система

```css
--bg:         #f7f6f3   /* фон страницы, тёплый */
--white:      #ffffff   /* карточки */
--border:     #e4e1db   /* все разделители */
--border-2:   #d0cdc6   /* активные границы */
--text:       #1a1917   /* заголовки */
--text-2:     #52504a   /* основной текст */
--muted:      #9b9890   /* подписи, лейблы */
--accent:     #1d4ed8   /* один цвет — все CTA */
--accent-bg:  #eff4ff   /* фон акцентных элементов */
```

Цвет акцента используется строго для:
- Кнопка primary
- Ключевое слово в заголовке
- Активные состояния
- Прогресс-бары и индикаторы

---

## Навигация

Высота 56-60px. Три зоны: логотип / ссылки / кнопки. Backdrop-filter blur при скролле. Никаких выпадающих меню на лендинге — ведут в никуда и отвлекают.

Логотип: цветной квадрат 26-28px с иконкой + название. Никаких сложных лого.

## CTA секция в конце

Не повторяйте весь лендинг. Одна карточка на акцентном фоне:

```
Короткий заголовок (призыв)
Одна строка про условия (бесплатно / без карты)
Поле email + кнопка
Маленькая подпись с цифрой доверия
```

## Что никогда не делать

- Стоковые фотографии людей за ноутбуками
- Анимированные иконки которые крутятся сами по себе
- Больше двух шрифтов
- Градиентный текст на градиентном фоне
- Секция «Наши партнёры» с серыми логотипами
- Карусели и слайдеры
- Всплывающие окна в первые 5 секунд
- Счётчик обратного отсчёта («предложение истекает через»)
- Тёмная тема с фиолетовым градиентом — это уже клише

## Технический стек

Один HTML файл. Google Fonts. Чистый JS без jQuery. CSS переменные для всей системы. Intersection Observer для появления блоков при скролле. Никаких библиотек анимаций — всё через CSS transition и @keyframes.

Это быстро грузится, легко редактируется и не ломается.
