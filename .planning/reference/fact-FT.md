# Функциональные требования: fact.getgantt.ru

## 1. Общие сведения
- **Назначение:** Отдельный мобильный интерфейс для ежедневного сбора факта выполнения работ на строительной площадке.
- **Целевая аудитория:** Прорабы, мастера участков, бригадиры, субподрядчики.
- **Ключевой принцип:** Минимум полей, крупные элементы, пошаговый сценарий. Прораб не видит весь график, только свои задачи.
- **Связка:** Данные автоматически передаются в основной `getgantt.ru` для формирования слоя план-факт, аналитики отклонений и прогнозов.

## 2. Авторизация и видимость данных
- Вход по номеру телефона / одноразовой ссылке / корпоративному аккаунту.
- **Фильтрация работ:** Прораб видит только задачи, где он является ответственным, закреплен за локацией, или относится к назначенной бригаде/подрядчику.
- Доступ ограничивается уровнем проекта → корпуса → секции → участка.

## 3. Навигация и структура интерфейса
- 4 фиксированные вкладки внизу: `Сегодня`, `Объект`, `Проблемы`, `Журнал`.
- Горизонтальный переключатель дней:Yesterday/Today/Tomorrow + скролл на неделю.
- Экран `Сегодня` открывается по умолчанию.

## 4. Экран «Сегодня» (план-факт)
- Отображение списка карточек работ, запланированных на выбранный день.
- **Карточка работы содержит:**
  - Название задачи
  - Локация (Корпус → Секция → Этаж)
  - План объема (ед. измерения) или отметка «без единиц»
  - Назначенная бригада
  - Плановые сроки
  - Текущий прогресс (полоса + %)
  - Цветной статус-бейдж (по плану / риск / проблема / завершена / не начата)
- **Быстрые действия на карточке:** `+ Факт`, `Проблема`, `Фото`, `Не работали`.
- Баннер `Закрытие дня` со сводкой: всего по плану / отмечено / не отмечено / с проблемами.

## 5. Ввод факта (модальный экран)
- **Выбор статуса:** `Выполнялась`, `Не выполнялась`, `Завершена`, `Есть проблема`.
- **Ввод объема:**
  - Поле `Сделано сегодня` (в единицах измерения, если заданы в плане).
  - Автоматический расчет `Всего выполнено` и `%`.
  - Альтернативный режим: крупные кнопки прогресса `0% · 25% · 50% · 75% · 100%`.
- **Учет ресурсов:**
  - Переключатель `Работали / Не работали`.
  - Поле `Кол-во человек`.
  - Подтянутая из плана бригада (с возможностью замены на фактическую).
- **Фото:** Загрузка 1+ изображений. Автоматическая привязка метаданных (проект, задача, дата, локация, автор).
- **Комментарий:** Свободное текстовое поле.
- **Отправка:** Кнопка `Отправить факт` с валидацией минимально необходимых полей.

## 6. Управление проблемами
- Отдельная вкладка `Проблемы` со списком открытых и решенных.
- **Карточка проблемы:** Название/привязанная задача, локация, причина (тег), дата/время, автор.
- **Создание проблемы:**
  - С привязкой к конкретной работе.
  - Без привязки (свободная заявка, позже привязывается РП).
- **Выбор причины:** Готовые теги: `Нет материала`, `Нет людей`, `Не готов фронт`, `Ждём смежников`, `Изменение проекта`, `Погода`, `Техника`, `Другое`.
- Возможность приложить фото и текстовое описание.
- Изменение статуса проблемы пользователем (открыта / решено / ожидает реакции РП).

## 7. Закрытие дня
- Режим вечерней сверки для предотвращения пропусков.
- Сводка: количество работ по плану vs фактически отмеченных.
- Список неотмеченных работ с быстрыми вариантами реакции:
  - `Внести факт` (переход к форме ввода)
  - `Не работали` (автозаполнение статуса)
  - `Перенести` (формирование запроса на корректировку сроков)
  - `Пропустить`
- Кнопка `Закрыть день` фиксирует состояние суток и блокирует редактирование прошлого дня (кроме правок с пометкой).

## 8. Экран «Объект» (навигация по структуре)
- Иерархическое дерево: `Объект → Корпус → Секция → Этаж → Зона/Помещение → Работа`.
- Раскрытие/сворачивание узлов.
- Отображение количества активных задач на каждом уровне.
- Блок `Завтра по плану` для предварительного ознакомления с фронтом работ.
- Позволяет прорабу, находясь на локации, быстро отфильтровать работы по месту, а не по дате.

## 9. Журнал факта
- Хронологическая лента всех отправленных записей.
- Группировка по датам.
- **Запись содержит:** цветной индикатор статуса, название задачи, локация, время отправки, внесенные данные (объем, %, бригада), миниатюры фото, комментарий.
- Статус синхронизации: `Отправлено` / `Принято графиком` / `Требует уточнения`.
- Цель: прозрачность для исполнителя («я отправил, всё зафиксировано»).

## 10. Интеграция с основным GetGantt
- **Передача данных:** Прогресс задачи, дата последнего обновления, история изменений, фото, открытые проблемы, фактическое кол-во людей.
- **Визуализация в основном графике:**
  - Слой факта поверх плановой полосы.
  - Цветовая индикация: `🟢 по плану`, `🟡 риск`, `🔴 отставание`, `⚪ факт не подан`, `🟣 завершено досрочно`.
  - Иконки-счетчики на задаче: `📷 N фото`, `⚠️ M проблем`.
- **Аналитика для РП:**
  - Агрегация причин отставания по проекту.
  - Сравнение план/факт освоения объемов.
  - Формирование прогноза завершения на основе фактической производительности.

## 11. Технические и UX-требования
- Mobile-first, адаптив до 480px, поддержка iOS/Android браузеров.
- Поддержка PWA (установка на домашний экран, базовый кэш для просмотра последнего состояния).
- Время заполнения одной карточки факта: ≤ 2 минут.
- Автосохранение черновика при потере связи или уходе с экрана.
- Все поля с выбором значений используют крупные кнопки/чипы, минимизация ручного ввода текста.

## 12. Границы MVP (исключено)
- Табель учета рабочего времени, КС-2/КС-3, акты освидетельствования.
- Чек-листы качества, замечания технадзора, предписания.
- Встроенный мессенджер, комментарии между пользователями.
- QR/NFC-маркировка зон.
- Полноценный офлайн-режим с синхронизацией очередей.
- Сложная матрица ролей и маршрутизации согласований.
