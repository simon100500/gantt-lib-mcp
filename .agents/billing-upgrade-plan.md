### A6. Решить конфликт: счётчик на billing page

pricing-context.md: "Лимиты скрыты (нет счётчиков в интерфейсе)".
AccountBillingPage показывает progress bar `aiUsed / aiLimit`.

**Предложение:** оставить счётчик на billing page (это страница управления подпиской, пользователь сам туда заходит). Убрать/не показывать счётчик в основном интерфейсе приложения (редактор, sidebar). Принцип "скрыты" относится к рабочему экрану, а не к биллингу.

Требует подтверждения.

## Часть B: CRO-улучшения upgrade flow

### B1. Feature gate modal при достижении лимита (HIGH IMPACT)

**Что:** модальное окно, когда пользователь пытается выполнить AI-запрос / создать проект, но упёрся в лимит.

**Где:** новый компонент `LimitReachedModal.tsx` (или inline в месте вызова AI/создания проекта).

**Контент (из pricing-context.md):**

Для AI-лимита (free — закончились 20 запросов):
```
Вы сделали много изменений сегодня.
Снимите ограничения и продолжайте работу.

[Обновить тариф]     [Не сейчас]
```

Для AI-лимита (платный — дневной лимит):
```
Вы сделали много изменений сегодня.
Лимит обновится завтра в 00:00.

[Расширить тариф]    [Понятно]
```

Для лимита проектов (free — 1 проект):
```
Чтобы создать новый проект, освободите место
или расширьте тариф.

Текущий: 1 из 1 проекта

[Расширить тариф]    [Удалить проект]
```

**Принципы из pricing-context:**
- Формулировки мягкие, без "лимит исчерпан"
- Апсейл через ценность, а не через ограничения
- Нет резкого обрыва — предупреждение перед стопом (фаза 2)

**Файлы для изменения:**
- Новый: `packages/web/src/components/LimitReachedModal.tsx`
- Изменить: место, где вызывается AI (нужно найти — вероятно store или API-слой)
- Изменить: место создания проекта (аналогично)

### B2. Контекстный апсейл в AccountBillingPage при высоком usage

**Что:** alert-блок, когда AI usage >= 80%.

**Где:** `AccountBillingPage.tsx`, внутри блока progress bar.

**Контент:**
```
Осталось мало запросов.
Расширьте тариф, чтобы не останавливать работу.

[Расширить → Старт за 1 490 ₽/мес]
```

При 100%:
```
AI-запросы закончились.
Обновите тариф для продолжения работы.

[Обновить тариф]
```

**Файлы:** `AccountBillingPage.tsx` — добавить условный блок после progress bar.

### B3. Персонализация кнопки "Расширить" в AccountBillingPage
